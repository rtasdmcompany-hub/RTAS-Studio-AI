## 3. Partner categories (realistic)

| Category | Why relevant | What RTAS offers | What RTAS needs |
|----------|--------------|------------------|-----------------|
| **Creative / digital agencies** | Recurring video demand; client trust review | Credits clarity, trust pages, studio workflow | Pipeline of campaigns; case-study permission later |
| **Production boutiques & editors** | Overflow capacity; hybrid AI + craft | Generation for look-dev / variants | Quality feedback; referral |
| **Marketing tech / DAM / project tools** | Workflow adjacency | Embeddable studio narrative (when API ready) | Distribution inside existing tools |
| **Model / infra providers** | Capability backbone | Packaged commercial UX + support story | Reliable inference, commercial terms |
| **Education / creator schools** | Train next users | Student Tester paths, curriculum hooks | Authentic student work (authorized) |
| **Regional GTM resellers** | Local language/sales in MEA / South Asia / EU | Product + MoR professionalism | Compliant local selling; no AUP violations |
| **Payment / commerce ecosystem** | Checkout trust | Clean AUP product story | Working MoR approval and settlement |
| **Complementary SaaS** (stock, music, design) | Bundle creative stack | Co-marketing when mature | Non-conflicting brand fit |

**Out of scope:** Any partner whose primary pitch is unauthorized likeness, non-consensual deepfakes, or “undetectable” impersonation.

## 4. Selection criteria

Score candidates 1–5 on each; pursue only if **AUP Fit ≥ 5** (binary gate) and total ≥ 18/25 on remaining.

| Criterion | Question |
|-----------|----------|
| **AUP Fit (gate)** | Will they refuse deepfake/face-swap marketing of RTAS? |
| Audience overlap | Do they reach creators, agencies, or marketing teams in ICP? |
| Brand safety | Would their logo next to RTAS survive enterprise legal review? |
| Commercial clarity | Can economics be simple (referral %, seat bundles, or integration fee)? |
| Execution capacity | Is there a named owner on their side within 30 days? |
| Proof path | Can we get a permissioned pilot story within 90 days? |
| Technical readiness | If integration: is scope honest relative to current API maturity? |
| Geo / compliance | Any sanctions, IP, or privacy red flags? |

| Model | Description | Stage fit |

## 6. Outreach sequence (founder-executable)

1. Identify 20 targets across agencies + one tech adjacency.  
2. Personalized note: studio use cases + AUP stance + Tester invite.  
3. 30-minute pilot scoping call (use enterprise demo script patterns from Sprint 4 sales docs).  
4. Written pilot goals (deliverable count, credit budget, success metric).  
5. Only then discuss co-marketing or referral terms.  
6. Public logo use requires written approval from both sides.

Contacts: contact@rtasstudio.com · info@rtasstudio.com

## 7. Risk controls

| Risk | Mitigation |
|------|------------|
| Partner markets RTAS as deepfake | Immediate termination clause; public correction |
| Partner invents RTAS metrics | Provide approved fact sheet only |
| Traffic before checkout works | Prioritize MoR reliability; soft-launch partners |
| Exclusive demands too early | Prefer non-exclusive pilots |
