# Retention Strategy

**Product:** RTAS Studio AI  
**Phase:** 13 · Sprint 6  
**Surface:** `/retention` (+ `/engage`, `/referral`, `/updates`)

---

## Goals

1. Reach first completed video quickly.  
2. Create a second project within 14 days.  
3. Convert Tester → Standard/Premium before the 5-day window ends.  
4. Keep paid accounts generating within each billing cycle.

## Product levers

| Lever | Implementation |
|-------|----------------|
| Usage insights | Real 14-day generation / credit trend |
| Upgrades | Pricing CTAs by current tier |
| Feature discovery | Studio, long-form stitch, Premium 4K, changelog |
| Learning | How-to, Help, FAQ, showcase |
| Referral | Sprint 4 `StudioReferralCode` (proposed rewards) |
| Milestones | Verified, project, video, paid, recent usage |
| Engagement Center | `/engage` from Sprint 3–4 |

## Integrity

- Referral metrics start at zero until real invites.  
- No fabricated retention rates in UI or docs.  
- Churn tips on Retention Center are rule-based (see `CHURN_PREVENTION.md`).
