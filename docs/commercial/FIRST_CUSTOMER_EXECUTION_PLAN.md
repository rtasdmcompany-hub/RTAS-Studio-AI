# First Customer Execution Plan

**Product:** RTAS Studio AI · `https://rtasstudio.com`  
**Phase:** 12 · Sprint 7  
**Integrity:** No fake customers, reviews, testimonials, or revenue.

## Reality check

This plan is **execution design**, not a claim that those user counts already exist. Targets below are operational milestones for the founder/ops team.

## Milestone ladder

### 10 users (prove the loop)

**Goal:** 10 real accounts that complete Signup → Verify → Dashboard → Studio → at least one render attempt.

| Focus | Actions |
|-------|---------|
| Channels | Personal outreach, beta applications (`/beta`), warm agency contacts |
| Product | Fix onboarding friction; watch `/profile` checklist completion |
| Sales | Offer Tester ($5) as evaluation; log every support reply |
| Success metric | ≥7/10 reach First Video or Credits Used on checklist |

**Do not:** publish logos, invented NPS, or “thousands of creators.”

### 100 users (stabilize retention)

**Goal:** Repeatable weekly acquisition without inventing growth numbers.

| Focus | Actions |
|-------|---------|
| Channels | Content (how-to, showcase), `/feedback` loops, partner intros (`/partners`) |
| Product | Reduce verify/email friction; clear upgrade CTAs when credits = 0 |
| Sales | Route serious teams to `/demo` / `/enterprise` |
| Success metric | Weekly active renders; support SLA under 2 business days |

### 500 users (systematize)

**Goal:** Documented ops for inbound + paid conversion (when Paddle MoR live).

| Focus | Actions |
|-------|---------|
| Channels | SEO pages already indexed; consistent Help/changelog |
| Sales | Pipeline stages from `ENTERPRISE_SALES_PIPELINE.md` |
| Product | Capacity monitoring (GPU wallet, status page) |
| Success metric | Paid Tester/Standard conversions from real checkout only |

### 1000 users (scale carefully)

**Goal:** Capacity and support before aggressive spend.

| Focus | Actions |
|-------|---------|
| Channels | Paid acquisition only after unit economics measured on real payments |
| Enterprise | Dedicated demo/quote capacity; proposal template outside app |
| Risk | GPU cost, abuse on free preview, support load |
| Success metric | Contribution margin positive on Standard/Premium cohorts |

## First-customer journey (product)

```
Landing → Signup → Verify email → /profile?welcome=1
  → Getting started checklist → Studio → First export
  → Upgrade prompt (paywall / Dashboard plans) when credits insufficient
```

## Weekly founder checklist

1. Review new beta + enterprise + demo emails (no fake backlog).  
2. Close one onboarding friction item from real user reports.  
3. Reply to `/feedback` submissions.  
4. Confirm email delivery (`RESEND_API_KEY` / domain) still healthy.  
5. Never invent metrics for investors or marketing.
