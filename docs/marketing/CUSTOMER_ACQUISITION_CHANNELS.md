# RTAS Studio AI — Customer Acquisition Channels

**Classification:** Marketing / demand gen  
**Phase:** 12 · Sprint 5  
**Integrity:** Pros/cons/effort/KPI **frameworks** only. No invented CAC, volume, or ROI numbers.

Cross-links: [`../growth/CUSTOMER_ACQUISITION_STRATEGY.md`](../growth/CUSTOMER_ACQUISITION_STRATEGY.md) · [`MARKETING_KPI_FRAMEWORK.md`](MARKETING_KPI_FRAMEWORK.md)

---

## Acquisition principle

**Pay-to-prove.** Entry offer = **Tester ($5 / 30s / 5d)**. Free accounts have **0 credits**. Do not scale paid channels until Paddle checkout is reliable.

---

## Channel matrix

### 1. SEO

| | |
|--|--|
| **Pros** | Compounding; high intent; fits credit/pricing clarity content |
| **Cons** | Slow; competitive SERPs for “AI video” |
| **Effort** | Medium–High ongoing |
| **Stage fit** | Always-on from day one |
| **KPIs (formulas)** | Organic sessions; GSC clicks; signup rate from organic landing pages; Tester starts attributed |

### 2. Content marketing

| | |
|--|--|
| **Pros** | Educates credit model + trust; fuels SEO/social/email |
| **Cons** | Needs consistency; blog not full CMS yet |
| **Effort** | Medium |
| **Stage fit** | Always-on |
| **KPIs** | Content→session; assisted signups; time-to-first-render for readers who convert |

### 3. Organic social

| | |
|--|--|
| **Pros** | Low cash cost; brand; Showcase distribution |
| **Cons** | Algorithm volatility; time-heavy |
| **Effort** | Medium |
| **Stage fit** | Always-on |
| **KPIs** | Profile link CTR; landing sessions from social; follower growth **only if measured** (never fabricate) |

### 4. Paid search

| | |
|--|--|
| **Pros** | High intent; controllable |
| **Cons** | Costly; needs conversion tracking; waste if checkout broken |
| **Effort** | Medium ops + budget |
| **Stage fit** | After MoR green |
| **KPIs** | `CAC = ad_spend / paying_customers`; CTR; CVR landing→Tester; refund rate |

### 5. Paid social

| | |
|--|--|
| **Pros** | Creative testing; retargeting visitors |
| **Cons** | Lower intent; brand-safety review needed (no deepfake ads) |
| **Effort** | Medium |
| **Stage fit** | After MoR green |
| **KPIs** | CAC; CPM/CPC (from ads manager); Tester CVR; frequency caps |

### 6. Affiliate

| | |
|--|--|
| **Pros** | Performance-aligned; extend reach via educators/agencies |
| **Cons** | Not live; payout/compliance work; fraud risk |
| **Effort** | High to launch |
| **Stage fit** | Proposed until MoR + payouts ready |
| **KPIs** | Affiliate-attributed paying accounts; payout / revenue; fraud rate |

### 7. Referral

| | |
|--|--|
| **Pros** | Trust transfer; low creative cost |
| **Cons** | Needs productized referral (if/when built); until then founder-led only |
| **Effort** | Low–Medium |
| **Stage fit** | Soft (ask happy users) → later formal program |
| **KPIs** | Referred signups; referred paid; reward cost (when exists) |

### 8. Influencer / creator education

| | |
|--|--|
| **Pros** | Demo authenticity; music/commercial niches |
| **Cons** | Cost; disclosure; quality variance; no fake endorsements |
| **Effort** | Medium–High per deal |
| **Stage fit** | Selective after checkout green |
| **KPIs** | Code/link conversions; CAC; brand-safety incidents (target = 0) |

### 9. Communities (Discord, Reddit, forums)

| | |
|--|--|
| **Pros** | Retention + authentic Q&A; ICP density |
| **Cons** | Spam backlash if self-promo heavy |
| **Effort** | Low–Medium continuous |
| **Stage fit** | Always-on |
| **KPIs** | Referral sessions; support deflection; qualitative feedback |

### 10. Enterprise outreach

| | |
|--|--|
| **Pros** | Higher ACV potential; brand-safety buyers |
| **Cons** | Long cycle; product may lack SSO/multi-seat today — be honest |
| **Effort** | High touch |
| **Stage fit** | Parallel founder-led |
| **KPIs** | Accounts contacted; meetings; pilots; closed-won (CRM only) |

---

## Recommended sequence

1. Organic SEO + content + How-to/Help accuracy  
2. Founder outbound (agencies)  
3. Organic social + Discord  
4. Paid search/social **only after** checkout + analytics  
5. Affiliate / formal referral when ops ready  
6. Enterprise expansion with truthful packaging  

---

## Shared attribution rules

- Prefer first-touch + last-touch notes in CRM; do not invent multi-touch percentages.  
- Tester revenue is one-time — report separately from MRR.  
- Pause any paid channel if checkout failure rate or refund confusion spikes.
