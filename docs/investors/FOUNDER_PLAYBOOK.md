## 1. Mindset

1. **Credibility compounds; fiction detonates.**  
2. MoR status is a first-class talking point—never bury it.  
3. You are early-stage with a productized studio—own it.  
4. Dilution is optional; bad terms are worse than no round.  
5. AUP / Trust & Safety are non-negotiable growth hacks to refuse.

## 2. Investor meetings

### 2.1 Prep (48 hours)

| Prep item | Source |
|-----------|--------|
| 18-slide narrative | `PITCH_DECK_CONTENT.md` |
| One-page truth | `docs/sales/Executive-Summary.md` |
| MoR status sentence | Founder fact-check that morning |
| Demo path rehearsed | Studio happy path < 5 minutes |
| Q&A refresh | `INVESTOR_QA.md` |
| Ask framework | Only if raising; else “exploring / not raising yet” |

### 2.2 Meeting arc (30–45 min)

| Minutes | Segment |
|--------:|---------|
| 0–2 | Context: who you are, operator, Pakistan + global SaaS |
| 2–8 | Problem → Solution → live or recorded demo |
| 8–15 | Business model + pricing truth |
| 15–22 | Market (labeled) + GTM + competition honesty |
| 22–28 | Traction honesty + roadmap |
| 28–35 | Ask / process / next step |
| 35–45 | Q&A |

### 2.3 Rules in the room

- If you do not know a metric, say **“not yet measured”**  
- Never invent logos  
- Offer data room follow-up instead of guessing  
- Take notes on investor objections—they are product input  

### 2.4 Follow-up (same day)

1. Thank-you email with 3 links max (site, memo, deck content)  
2. MoR status if it changed  
3. Clear next step (second meeting, diligence folder, pass)

## 3. Negotiation basics

### 3.1 What you are negotiating

| Lever | Meaning |
|-------|---------|
| Valuation / cap | Price of equity or SAFE cap |
| Amount | Cash in |
| Dilution | Ownership given |
| Pro-rata / rights | Future round rights |
| Board seats | Control vs advice |
| Information rights | Reporting burden |
| Liquidation preference | Exit waterfall |
| Vesting / founder | Alignment |
| Exclusivity / no-shop | Opportunity cost |

### 3.2 Founder principles

- Prefer clean terms over maximal headline valuation  
- Avoid personal guarantees you do not understand  
- Multiple interested parties beat one desperate close—**without fake FOMO**  
- Walk away from pressure to violate AUP for “growth”  

### 3.3 Saying no

Templates:

- “We’re not a fit if the thesis requires unauthorized likeness features.”  
- “We won’t provide projected MRR as if it were historical.”  
- “We need counsel review before signing a no-shop longer than [X] days.”

## 4. Term sheets (process — counsel-owned)

### 4.1 Before you request a term sheet

- [ ] MoR status current  
- [ ] Cap table draft ready under NDA  
- [ ] Use-of-funds story labeled as plan  
- [ ] Counsel identified  

### 4.2 When a term sheet arrives

1. Do **not** countersign emotionally the same hour  
2. Send to counsel  
3. Build a comparison grid (economic + control terms)  
4. List must-fix vs nice-to-have  
5. Negotiate in writing  
6. Only then circulate signature pages  

### 4.3 Instruments (educational)

| Instrument | Typical early use | Watch-outs |
|------------|-------------------|------------|
| SAFE | Angel / pre-seed speed | Cap, discount, MFN, post-money vs pre |
| Convertible note | Debt-like | Interest, maturity, security |
| Priced equity | Seed+ | Preference, board, protective provisions |

**This playbook does not provide legal templates.** Counsel drafts.

## 5. Due diligence (founder side)

### 5.1 Running a clean process

1. Use `DUE_DILIGENCE_CHECKLIST.md` as index  
2. Single folder share with permissions  
3. Name an owner for Q&A turnaround (48–72h target)  
4. Log every request / answer  
5. Redact secrets; never send `.env`  

### 5.2 Common diligence traps

| Trap | Fix |
|------|-----|
| Contradictory MoR statements | One status doc |
| Pricing mismatch across pages | Align to verified tiers |
| “Customers” without proof | Remove |
| Security overclaim | Point to actual docs only |
| Open tabs forever | Set process end date |

### 5.3 Technical diligence tips

- Offer read-only architecture walkthrough  
- Show job lifecycle and credit guards conceptually  
- Disclose Fal dependency honestly  
- Share known limitations proactively—builds trust

## 6. Acquisition inbound

### 6.1 When someone says “we might acquire you”

1. Stay calm; ask what strategic value they see  
2. Do not email financial fantasies  
3. Share public materials first  
4. NDA before deep financials / cap table  
5. Counsel early if seriousness rises  
6. Continue building product—deals die often  

### 6.2 Evaluation grid (founder)

| Factor | Questions |
|--------|-----------|
| Strategic fit | Do they want studio + trust or just talent? |
| Price / structure | Cash vs stock; earnout realism |
| Team future | Roles, location, retention |
| Product destiny | Maintain brand vs shut down? |
| AUP ethics | Will they force policy violations? |
| Certainty | Funding, approvals, timeline |

Frameworks: [`docs/sales/Acquisition-Memo.md`](../sales/Acquisition-Memo.md) · outreach: [`Acquisition-Outreach-Strategy.md`](../business/sales/Acquisition-Outreach-Strategy.md)

### 6.3 Exclusivity

Grant short, counsel-reviewed exclusivity only with clear diligence milestones and a kill date. Long no-shops without deposit/commitment are usually unfavorable for an early operator.
